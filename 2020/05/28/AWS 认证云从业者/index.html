AWS Certified Cloud Practitioner 考试为那些具备有效展示对 AWS 云的全面理解所必需的知识和技能的个人而设立，独立于其他 AWS Certification 所涉及的特定技术角色。考生可以在考试中心参加考试，也可以采用在线监考考试形式，在家中或办公室中舒适便捷地参加考试。

IAM(Identity and Access Management)

AWS Identity and Access Management (IAM) 是一种 Web 服务，可以安全的控制用户对AWS各种组件的访问权限。IAM可以控制哪些人用哪些方式使用哪些AWS资源。

功能特性

  • 账户内共享访问权限
    授予契特人管理和使用你的AWS账户资源的权限，而无需共享密码或访问密钥
  • 细化权限
    可以根据需要为每个用户的职责授权所需的不同权限
  • 对在EC2上运行的应用程序的AWS资源的安全访问
    IAM可以帮助提供在EC2实例上运行的应用程序访问其他AWS资源所需的临时凭证
  • 多重验证
    可以向你的账户和各个用户添加双重身份验证以实现更高安全性。借助MFA，你或你的用户不仅必须提供使用账户所需密码或访问密钥，还必须提供来自经过特殊配置的设备的代码。
  • 联合身份
    IAM允许用户通过提供临时凭证（例如通过自己公司网络或Google或facebook等身份验证）访问AWS资源，而无需用户拥有AWS账户
  • 安全的身份信息
    CloudTrail可用于接收日志记录，其中包含有关在账户中提出资源请求的人员的信息
  • PCI DSS合规性
    IAM支持由商家或服务提供商处理、存储和传输信用卡数据，而且已经验证符合支付卡行业（PCI）数据安全标准（DSS）
  • 与AWS服务集成
    IAM与大部分的AWS服务集成，为其他的AWS服务提供身份验证与授权服务
  • 最终一致性
    IAM通过在亚马逊全球数据中心内的多个服务器之间复制数据，最终保持一致并实现高可用性。对IAM所作的更改最终会保持一致，因此需要一些时间来反映。
  • 免费使用
    IAM免费提供。比如你在其他资源上使用IAM控制权限，所收取的费用只是收取其他资源的费用
  • 安全令牌服务(Security Token Service)
    IAM提供STS，这是AWS账户的附带功能，免费提供。AWS仅对使用AWS STS临时安全凭证访问的其他AWS服务收费

基本概念（Basic concept）

  • 用户
    IAM用户是在AWS中创建的实体，使用用户名和密码可用于登录AWS控制台
  • 组
    是IAM用户集合，可以将用户加入到组，并为组指定权限，从而使其更易于管理
  • 角色
    IAM角色类似于用户，但是不是用户，不能用来登录，一般被赋予某个资源，使它临时具备某些权限，角色在账户内是不分区域的，并且角色比起使用用户的秘钥来说更安全和更容易管理。你可以在任何时候将角色赋予EC2实例，并且不需要重启而能够马上生效。
  • 策略与授权
    策略与权限允许你定义谁可以访问什么资源，通过策略文档描述具体的权限设置，策略描述文件为JSON

身份验证（Authentication）

  • AWS CLI
  • AWS SDK
  • AWS Management Console

EC2（Elastic Compute Cloud）

Amazon EC2是一种Web服务，向用户提供虚拟化服务器(虚拟计算环境)，被称为实例，它允许用户通过配置自动按需调整计算容量，例如可以对实例数量、CPU数量、内存大小、存储大小与类型以及网络性能进行弹性缩放。
Elastic是指如果配置正确，可以根据业务需求自动增加或减少所需的服务器数量，Compute是指计算服务器资源，Cloud是这些计算资源是由云托管。

EC2的价格模型

Amazon EC2 提供免费试用。Amazon EC2 实例有五种付费方式：按需实例、Savings Plans、预留实例、Spot 实例，以及专用主机。专用主机可在专供您使用的物理服务器上提供 EC2 实例容量。

按需实例

借助按需实例，您可以根据运行的实例以按小时或按秒的方式为计算容量付费。无需签订长期合同或支付预付款。您可以根据应用程序的需求提升或降低计算容量，并只需按规定的每小时费率为所使用的实例付费。

按需实例推荐用途 ：

  • 希望拥有低成本和Amazon EC2提供的灵活性，且不想支付预付款或签订长期合同的用户
  • 具有短期、难应付或无法预测且不能中断的工作负载的应用程序
  • 首次在Amazon EC2上开发或测试的应用程序

Savings Plans

Savings Plans 是一种灵活的定价模式，它以较低的 EC2 和 Fargate 使用价格，换取在 1 年或 3 年期限内的稳定使用量承诺（以 USD/小时为单位衡量）。

预留实例

可以提供容量预定，并且与按需实例相比，提供大幅折扣，合同期为1或者3年

预留实例推荐用途:

  • 使用率稳定的应用程序
  • 可能需要预留容量的应用程序
  • 愿意使用EC2 1年或3年以上来降低总计算成本的客户

预留实例类型：

  • 标准预留实例：与按需实例的定价相比，预留实例可提供折扣高达75%，并能按1年或3年的使用期进行购买。客户能够灵活的更改其标准预留实例的可用区、实例大小以及联网类型。
  • 可转换预留实例：如果需要额外的灵活性（例如能够在预留实例使用期内使用不同的实例系列、操作系统或租期），可以购买可转换预留实例。与按需实例的定价相比，可转换预留实例可提供高达54%的折扣，并能按1年或3年的使用期进行购买。
  • 计划预留实例：可以以一年为期限购买具有指定的开始时间和使用时长。需要提前预定预留容量，以确定在需要的时间可用，一旦购买，即使你未使用也需要付费。

Spot 实例（竞价实例）

可以以任何价格竞拍实例，与按需实例相比，这类实例最多可以节省90%的成本

Spot实例推荐用途：

  • 开始时间或结束时间灵活的应用程序
  • 只能承受极低的计算价格的应用实例
  • 具有紧急计算需要，需要获取大量附加容量的用户

专用主机

专用主机是指专攻你使用的物理EC2服务器，专用主机可以用来绑定软件许可证

专用主机推荐用途：

  • 用于满足不支持多租户虚拟化的法规需求
  • 可按需购买（按小时）
  • 可作为预留实例购买，与按需实例的价格相比，最多可以节省70%的成本
  • 非常适合不支持多租户或云部署的许可

专用实例与专用主机的区别：专用实例是在单一客户专用硬件上运行的Amazon EC2实例。专用主机与专用实例在性能、安全性或物理方面没有区别。不同的是，使用专用主机，可以查看和控制实例在服务器中的放置。

EC2的实例类型

安全组

安全组 充当实例的虚拟防火墙，可以控制传入和传出流量。入站规则控制传入到实例的流量，出站规则控制从实例传出的流量。

EBS(Elastic Block Store)

可扩展块存储设备，我们可以利用EBS创建存储卷并挂载到EC2实例上，挂载之后可以在存储卷上创建文件习题，运行数据库或其它用途。EBS卷可以放置在一个指定的可用区，我们可以设置自动复制来避免EBS卷损坏导致数据丢失。

EBS的特点

  • 挂载在EC2上面的持久的、自定义块存储设备
  • 同可用区复制
  • 通过快照备份
  • 简单容易加密
  • 可扩展

EBS类型

ELB(Elastic Load Balancing)

Elastic Load Balancing 在多个目标（如 Amazon EC2 实例、容器、IP 地址和 Lambda 函数）之间自动分配传入的应用程序流量。它可以在单个可用区内处理不断变化的应用程序流量负载，也可以跨多个可用区处理此类负载。Elastic Load Balancing 提供三种负载均衡器，它们均能实现高可用性、自动扩展和可靠的安全性，因此能让您的应用程序获得容错能力。

ELB的类型和特点

传统负载均衡(Classic Load Balancer) 2009年发布的第一代负载均衡
位于开放系统互联(OSI)模型的第四层，支持静态IP用于负载均衡

应用负载均衡(Application Load Balancer) 2016年发布的第二代负载均衡
位于应用程序层，也就是开发系统互联(OSI)模型的第七层，支持基于路径的路由和基于主机的路由

网络负载均衡(Network Load Balancer) 2017年发布的第二代负载均衡
位于开放系统互联(OSI)模型的第四层，能每秒处理上百万的网络请求，支持静态IP用于负载均衡

ELB的模式

面向公开网络
面向Internet的负载均衡器通过Internet从客户端接收请求，并将这些请求分发到使用负载均衡器注册的EC2实例

面对内部
对接内部网络，从外部不可访问

Autoscaling

根据业务需求和伸缩策略，自动调整计算资源。

自动伸缩的好处：

  • 快速部署新的应用程序
  • 随工作负载增加而即时扩展
  • 即时关闭无用的资源
  • 缩减时，无需再为相应的基础设施付费

自动伸缩(cloudwatch + autosacling + elastic load balance)

  • 垂直缩放(垂直扩展scale-up,垂直收缩scale-down)
    通过增加单个资源来进行，例如增加RAM，CPU，IOPS(磁盘每秒输出量)或者更新实例类型
  • 水平缩放(水平扩展scale-out,水平收缩scale-in)
    通过增加更多资源来实现，例如增加多个服务器，或者存储卷。

S3(Simple Storage Service)

AWS托管的云存储服务，可以存储无限数量的文件，随时随地的访问，丰富的权限控制。

主要特征：

  • 较低的延迟和较高的吞吐量性能
  • 可跨多个可用区实现99.999999999(11个9)的对象的持久性
  • 针对影响整个可用区的事件具有弹性
  • 可在指定年度内实现99.99%的可用性
  • 由Amazon S3服务等级协议提高支持，实现可用性
  • 支持传输中数据SSL和静态数据加密
  • 用于自动将对象迁移到其它S3存储类的S3生命周期管理

S3 Glacier

S3 Glacier是成本极低的存储服务，为数据存档和备份提供安全而持久的存储。为了降低成本，Amazon Glacier专门针对不经常访问的可以接受数小时检索时间的数据进行了优化。利用Amazon Glacier，客户可以长时间可靠的存储大量数据。

S3 存储类

  • S3 标准(S3 Standard)
    针对频繁访问的数据，S3 标准提供较高的持久性、可用性和性能对象存储。
  • S3 智能分层(S3 intelligent-Tiering)
    S3 智能分层存储类设计为通过自动将数据移至最经济高效的访问层，而不影响性能或运行开销来优化成本。它的工作原理是：将对象存储在两个访问层中：一个层已针对频繁访问而优化，另一个成本较低的层已针对不频繁访问而优化。
  • S3 标准-IA(S3 Standard-Infrequent Access)
    S3 标准 - IA 适用于不常访问、但在需要时要求快速访问的数据。
  • S3 单区-IA(S3 One Zone-Infrequent Access)
    S3 单区 - IA 适用于不常访问、但在需要时要求快速访问的数据。其他 S3 存储类将数据存储在至少三个可用区 (AZ) 中，而 S3 单区 - IA 将数据存储在单个 AZ 中并且成本较 S3 标准 - IA 低 20%。
  • S3 Glacier
    S3 Glacier 是安全、持久且成本低的存储类，可用于数据存档。
  • S3 Glacier深度存档(S3 Glacier Deep Archive)
    S3 Glacier Deep Archive 是 Amazon S3 成本最低的存储类，支持每年可能访问一两次的数据的长期保留和数字预留。

RDS

Amazon Relational Database Service (Amazon RDS) 让您能够在云中轻松设置、操作和扩展关系数据库。它在自动执行耗时的管理任务（如硬件预置、数据库设置、修补和备份）的同时，可提供经济实用的可调容量。
Amazon RDS 在多种类型的数据库实例（针对内存、性能或 I/O 进行了优化的实例）上均可用，并提供六种常用的数据库引擎供您选择，包括 Amazon Aurora、PostgreSQL、MySQL、MariaDB、Oracle Database 和 SQL Server。

可用性与持久性

Amazon RDS 让您能够轻松地使用复制功能来增强生产工作负载的可用性和可靠性。

  • 多可用区(Multi-AZ)
    使用多可用区部署选项，您可以执行关键任务工作负载，并且能够利用高可用性和内置的自动故障转移功能，从您的主数据库转移到同步复制的辅助数据库。

  • 只读副本
    只读副本可以轻松实现弹性扩展，超越单个数据库实例的容量限制，满足读取密集型数据库工作负载的需求。您可以为给定的源数据库实例创建一个或多个副本，利用多份数据副本满足大量应用程序读取流量需求，以此增加总读取吞吐量。

Aurora

高性能和可扩展性

  • 吞吐量
    根据 SysBench 等标准基准进行的测试表明，在类似硬件上运行时，Amazon Aurora 的吞吐量性能最高可达到存储 MySQL 的 5 倍，PostgreSQL 的 3 倍。
  • 存储自动扩展
    Amazon Aurora 会随着您数据库存储需求的增长而自动提高您的数据库容量大小。您的容量将以 10GB 的增量增加，最大可增加到 64TB。您无需为了满足未来增长需求，而为数据库预置多余存储空间。
  • 低延迟只读副本
    通过创建多达 15 个数据库 Aurora 副本，增加读取吞吐量以支持大量应用程序请求。
    Aurora 支持跨区域只读副本。跨区域副本为您的用户提供快速本地读取，并且每个区域可以具有 15 个额外 Aurora 副本以进一步扩展本地读取。

高可用性

  • 包含 Aurora 副本的多可用区部署
    发生实例故障时，Amazon Aurora 使用 RDS 多可用区技术，自动将故障转移到最多 15 个 Amazon Aurora 副本中的一个（您在 3 个可用区中的任意一个内创建的副本）。

  • 全球数据库
    对于全局分布式应用程序，您可以使用全局数据库，其中单个 Aurora 数据库可以跨多个 AWS 区域，以实现快速本地读取和快速灾难恢复。

自愈性和容错性

  • 容错和自我修复型存储
    Amazon Aurora 存储具有容错能力，能以透明方式应对多达两个数据副本的丢失，而不会影响数据库写入可用性，还能在不影响读取可用性的情况下应对多达三个副本的丢失。Amazon Aurora 存储还具有自我修复能力，可连续扫描数据块和磁盘是否存在任何错误，并自动更换。

DynamoDB

Amazon DynamoDB 是一个 NoSQL 数据库，它支持键-值和文档数据模型，并使开发人员能够构建无服务器的现代化应用程序，这些应用程序可以从小规模开始，然后扩展到全球范围以支持每秒数 PB 的数据和数千万条读取和写入请求。

  • NoSQL数据库表作为一种服务
  • 根据需要存储尽可能多的条目
  • 条目内容可以存储许多不同的属性
  • 支持文件和键值模型数据
  • 低延迟查询
  • 扩展读写吞吐量

适用场景：移动应用、web应用、游戏、广告、物联网（IoT）等程序中。

CloudFront

CDN(Content Delivery network)即内容分发网络，根据地理位置转送缓存页面和其它网络内容给客户，让用户能够更快的得到响应。
Amazon CloudFront是一个Web服务，它加快静态和动态Web内容(如.html、.css、.js和图像文件)分发到用户速度。CloudFront通过全球数据中心网络传输内容，这些数据中心称为边缘站点(边缘位置)。当用户请求用CloudFront提供的内容时，用户被路由到提供最低延迟(时间延迟)的边缘站点，从而以尽可能最快的速度响应用户请求。

关键术语

  • 边缘位置(Edge Location)：AWS用来缓存数据的终端节点，离用户最近位置的缓存内容的服务器群。不仅仅能够通过边缘位置读取内容，还能够通过边缘位置写入。
  • 内容源(Origin)：CloudFront分发的内容来源，可以是S3存储桶，ELB，MediaPackage源，MediaStore容器
  • 分配(Distribution)：当需要使用CloudFront分发内容的是时候，需要创建分配并选择所需要的设置：如设置内容源，访问权限，安全性，地理限制等。
  • TTL(Time to Life)：设置内容缓存在CloudFront的时长，超过这个时间，内容将被移除。
  • 分配的类别：
    • web：用来分发网页内容
    • RTMP：用来分发媒体流内容

CloudFormation

AWS CloudFormation可以模式化的创建AWS其他各种服务，让用户能花更少的时间去管理基础设施服务而能够更加专注在业务应用上。首先创建好包含需要的资源(例如EC2，RDS等)模板，然后CloudFormation使用模板来创建和配置资源组件。用户不需要单独挨个去创建和配置资源组件。

ELastic Beanstalk

通过Elastic Beanstalk,可以在AWS云中快速部署和管理应用程序，而不必了解运行这些应用程序的基础设施。ELastic Beanstalk可减少管理复杂性，而不会限制选择或控制权。用户只需上传应用程序，Elastic Beanstalk将自动处理有关容量配置、负载均衡、扩展和应用程序运行状况监控的部署细节。

  • 平台即服务
  • 快速部署应用
  • 减少管理复杂性

RedShift

Amazon RedShift是一种快速并且完全托管的云端大规模并行PB级数据仓库，可以让用户以轻松而经济高效的方式使用标准SQL和现有的商业智能工具对PB级的结构化数据进行大规模并行查询，并且在几秒钟内返回结果。
RedShift是一种联机分析处理OLAP(On-line Analytics Processing)的类型，支持复杂的分析操作，侧重决策支持，并且能提供直观易懂的查询结果。

数据处理大致分为两大类：

  • 联机分析处理OLAP:在线分析处理历史数据或档案数据。特点是交易量低、查询通常非常复杂。数据源通常存储在关系数据库的数据仓库中。
  • 联机事物处理OLTP(On-line Transaction Processing):也称为面向交易的处理系统，基本特征是顾客的原始数据可以立即传送到计算中心进行处理，并在很短的时间内给出处理结果，是传统关系型数据库的主要应用，主要是基本的、日常的事物处理，例如银行交易，在线电子商务系统等。

CloudWatch

CloudWatch是一个用来实时监控AWS服务状态和健康状况的监控服务，也是一个运行在aws上面的应用，可以用来监控运行在AWS上的其他应用。它能够提供性能以及运行监控状况的视图，通过跟踪，测量，报告，警告和重新激活事件提供系统层面的关于资源使用，应用性能和运行健康状况的视图。

组件：参数指标，警告，事件，，日志，面板

使用场景

  • 监控AWS服务状态变化
  • 使用CloudWatch事件记录S3对象级别操作日志
  • 根据计划对EBS卷做快照记录
  • 从CloudTrail到Kinesis流对指定API记录进行潜在的安全和可用性风险细节分析
  • 当EC2进入运行状态时通过CloudWatch事件提醒自动调用AWS Lambda功能更新DNS接口

CloudWatch日志

  • 使用日志文件对系统和应用进行监控和故障排除，可以对指定的阶段、值、字段进行监控
  • 从CloudWatch日志中取回辅助日志数据
  • 监控CloudTrail日志事件
  • 实时监控EC2实例日志

CloudTrail

AWS CloudTrail是一项对AWS账户进行监管、合规性检查、操作审核和风险审核的业务。通过CloudTrail，可以看到所有用户控制台活动记录，以及CLI、AWS开发包工具和API中执行的操作。相当于AWS账户的一个日志工具，会记录所有操作以用于审计。

使用场景

  • API调用：谁，什么时候，调用了什么，作用在哪些资源上，调用从哪里发起的(会清楚的记录IP地址信息)
  • 报警功能：结合CloudWatch抓取需要监控的关键词设置报警功能
  • 监控账户活动记录：通过控制台，命令行，开发包工具的活动记录，以及子账户的所有活动记录。

Route53

Amazon Route53是一种具有很高可用性和可扩展性的域名系统（DNS）Web服务。主要的三个功能：域名注册、DNS路由和运行状况检查。

DNS路由策略：

  • 简单路由
  • 地理位置路由
  • 故障转移路由
  • 加权路由
  • 基于延迟的路由
  • 多值应答路由

VPC(Virtyal Private Cloud)

VPC允许用户在已定义的虚拟网络内启用AWS资源。这个虚拟网络与在数据中心运行的传统网络极其相似，并会为你提供使用AWS的可扩展基础设置的优势。
每个账户在每个区域都会由一个默认的VPC，一个默认的VPC下面会有两个默认的子网。

ACL与EC2中的安全组的区别
ACL是无状态的，每一个入向流量和出向流量分别定义，并且ACL针对的是子网或者是VPC。
安全组是有状态的，定义入向流量之后出向流量自动打开，针对的是EC2实例。

标签、资源组与系统管理

标签
用来识别AWS云资源，将相同作用的资源归类，便于搜索和资源聚合管理。

资源组
相同特征的AWS服务设置同一种标签，然后根据标签创建一个组。

标签编辑器
可以给设置标签的资源重新编辑标签。

System Manager
简化运维、轻松大规模运行和管理基础设置资源

组织与合并支付

可以用来集中管理AWS的环境，例如管理账单、控制访问权限、合规性和安全性，并在多个用户之间共享资源。

主账户(根账户)

  • 在组织中为所有账户付款的账户
  • 管理组织的中心节点

组织单元(OU,organization unit)

  • 组织单元是组织内的一种逻辑组
  • 把AWS账户添加到逻辑组中，账号管理更方便
  • AWS账户和组织单元OU可以是另外一个组织单元PU的成员
  • 一个AWS账户可以是多个组织单元OU的成员

组织控制策略(Organization control policy(OCP))

  • 应用于组织单元和子账户的权限配置的策略

合并支付的好处：

  • 一个根账户负责支付所有子账户的费用
  • 集中付款容易控制成本，容易跟踪账单
  • 容易达到折扣额度

安全性

基础设施的安全性：合规性

账户的安全性：IAM（删除根用户密钥、激活根账户的多因子验证、创建单独的IAM用户、最小权限管理、密码策略）
服务的安全性：IAM role，secunity group（EC2实例级别），NACL，日志记录和监控、加密和密钥管理、网络分段、DDos防护(Shield)等核心服务
应用的安全性: WAF

责任共担模型


AWS负责“云的安全”，保护所有AWS云服务的基础设施的安全。

客户负责“云中的安全”，通过服务的配置、操作系统的更新、安全补丁的安装、安全组、网络控制列表NACL使用AWS提供的安全服务等措施来保护客户自己所选的AWS云服务以及使用过程中产生的数据安全和传输安全等。

AWS WAF & AWS Shield

AWS WAF 是一种 Web 应用程序防火墙，可帮助保护您的 Web 应用程序或 API 免遭常见 Web 漏洞的攻击，这些漏洞可能会影响可用性、损害安全性或消耗过多的资源。

AWS Shield 是一种托管式分布式拒绝服务 (DDoS) 防护服务，可以保护在 AWS 上运行的应用程序。AWS Shield 提供持续检测和自动内联缓解功能，能够尽可能缩短应用程序的停机时间和延迟，因此您不需要联系 AWS Support 来获得 DDoS 防护。AWS Shield 有两个层级，分别为 Standard(免费) 和 Advanced。

Trusted Advisor & AWS Inspector

AWS Trusted Advisor 是AWS提供的可帮助通过优化AWS环境降低成本、提高性能并改善安全性的在线工具，Trusted Advisor提供实时指导，帮助按照AWS最佳实践配置资源。Trusted Advisor 会从成本优化、性能、安全性、容错、服务限制提供建议。

Amazon Inspector 是一项自动安全评估服务，有助于提高在 AWS 上部署的应用程序的安全性与合规性。Amazon Inspector 会自动评估应用程序的风险、漏洞或者相较于最佳实践的偏差。执行评估后，Amazon Inspector 会生成按严重程度确定优先级的安全检测详细列表。
