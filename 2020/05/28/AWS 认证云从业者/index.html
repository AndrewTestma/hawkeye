IAM(Identity and Access Management)

AWS Identity and Access Management (IAM) 是一种 Web 服务，可以安全的控制用户对AWS各种组件的访问权限。IAM可以控制哪些人用哪些方式使用哪些AWS资源。

功能特性

  • 账户内共享访问权限
    授予契特人管理和使用你的AWS账户资源的权限，而无需共享密码或访问密钥
  • 细化权限
    可以根据需要为每个用户的职责授权所需的不同权限
  • 对在EC2上运行的应用程序的AWS资源的安全访问
    IAM可以帮助提供在EC2实例上运行的应用程序访问其他AWS资源所需的临时凭证
  • 多重验证
    可以向你的账户和各个用户添加双重身份验证以实现更高安全性。借助MFA，你或你的用户不仅必须提供使用账户所需密码或访问密钥，还必须提供来自经过特殊配置的设备的代码。
  • 联合身份
    IAM允许用户通过提供临时凭证（例如通过自己公司网络或Google或facebook等身份验证）访问AWS资源，而无需用户拥有AWS账户
  • 安全的身份信息
    CloudTrail可用于接收日志记录，其中包含有关在账户中提出资源请求的人员的信息
  • PCI DSS合规性
    IAM支持由商家或服务提供商处理、存储和传输信用卡数据，而且已经验证符合支付卡行业（PCI）数据安全标准（DSS）
  • 与AWS服务集成
    IAM与大部分的AWS服务集成，为其他的AWS服务提供身份验证与授权服务
  • 最终一致性
    IAM通过在亚马逊全球数据中心内的多个服务器之间复制数据，最终保持一致并实现高可用性。对IAM所作的更改最终会保持一致，因此需要一些时间来反映。
  • 免费使用
    IAM免费提供。比如你在其他资源上使用IAM控制权限，所收取的费用只是收取其他资源的费用
  • 安全令牌服务(Security Token Service)
    IAM提供STS，这是AWS账户的附带功能，免费提供。AWS仅对使用AWS STS临时安全凭证访问的其他AWS服务收费

基本概念（Basic concept）

  • 用户
    IAM用户是在AWS中创建的实体，使用用户名和密码可用于登录AWS控制台
  • 组
    是IAM用户集合，可以将用户加入到组，并为组指定权限，从而使其更易于管理
  • 角色
    IAM角色类似于用户，但是不是用户，不能用来登录，一般被赋予某个资源，使它临时具备某些权限，角色在账户内是不分区域的，并且角色比起使用用户的秘钥来说更安全和更容易管理。你可以在任何时候将角色赋予EC2实例，并且不需要重启而能够马上生效。
  • 策略与授权
    策略与权限允许你定义谁可以访问什么资源，通过策略文档描述具体的权限设置，策略描述文件为JSON

身份验证（Authentication）

  • AWS CLI
  • AWS SDK
  • AWS Management Console

EC2（Elastic Compute Cloud）

Amazon EC2是一种Web服务，向用户提供虚拟化服务器(虚拟计算环境)，被称为实例，它允许用户通过配置自动按需调整计算容量，例如可以对实例数量、CPU数量、内存大小、存储大小与类型以及网络性能进行弹性缩放。
Elastic是指如果配置正确，可以根据业务需求自动增加或减少所需的服务器数量，Compute是指计算服务器资源，Cloud是这些计算资源是由云托管。

EC2的价格模型

Amazon EC2 提供免费试用。Amazon EC2 实例有五种付费方式：按需实例、Savings Plans、预留实例、Spot 实例，以及专用主机。专用主机可在专供您使用的物理服务器上提供 EC2 实例容量。

按需实例

借助按需实例，您可以根据运行的实例以按小时或按秒的方式为计算容量付费。无需签订长期合同或支付预付款。您可以根据应用程序的需求提升或降低计算容量，并只需按规定的每小时费率为所使用的实例付费。

按需实例推荐用途 ：

  • 希望拥有低成本和Amazon EC2提供的灵活性，且不想支付预付款或签订长期合同的用户
  • 具有短期、难应付或无法预测且不能中断的工作负载的应用程序
  • 首次在Amazon EC2上开发或测试的应用程序

Savings Plans

Savings Plans 是一种灵活的定价模式，它以较低的 EC2 和 Fargate 使用价格，换取在 1 年或 3 年期限内的稳定使用量承诺（以 USD/小时为单位衡量）。

预留实例

可以提供容量预定，并且与按需实例相比，提供大幅折扣，合同期为1或者3年

预留实例推荐用途:

  • 使用率稳定的应用程序
  • 可能需要预留容量的应用程序
  • 愿意使用EC2 1年或3年以上来降低总计算成本的客户

预留实例类型：

  • 标准预留实例：与按需实例的定价相比，预留实例可提供折扣高达75%，并能按1年或3年的使用期进行购买。客户能够灵活的更改其标准预留实例的可用区、实例大小以及联网类型。
  • 可转换预留实例：如果需要额外的灵活性（例如能够在预留实例使用期内使用不同的实例系列、操作系统或租期），可以购买可转换预留实例。与按需实例的定价相比，可转换预留实例可提供高达54%的折扣，并能按1年或3年的使用期进行购买。
  • 计划预留实例：可以以一年为期限购买具有指定的开始时间和使用时长。需要提前预定预留容量，以确定在需要的时间可用，一旦购买，即使你未使用也需要付费。

Spot 实例（竞价实例）

可以以任何价格竞拍实例，与按需实例相比，这类实例最多可以节省90%的成本

Spot实例推荐用途：

  • 开始时间或结束时间灵活的应用程序
  • 只能承受极低的计算价格的应用实例
  • 具有紧急计算需要，需要获取大量附加容量的用户

专用主机

专用主机是指专攻你使用的物理EC2服务器，专用主机可以用来绑定软件许可证

专用主机推荐用途：

  • 用于满足不支持多租户虚拟化的法规需求
  • 可按需购买（按小时）
  • 可作为预留实例购买，与按需实例的价格相比，最多可以节省70%的成本
  • 非常适合不支持多租户或云部署的许可

专用实例与专用主机的区别：专用实例是在单一客户专用硬件上运行的Amazon EC2实例。专用主机与专用实例在性能、安全性或物理方面没有区别。不同的是，使用专用主机，可以查看和控制实例在服务器中的放置。
